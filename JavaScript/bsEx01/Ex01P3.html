Background

Although keeping the event listeners inside the JavaScript was an improvement over previous practices there are still limitations, for instance you can only have one Event Handler per listener.

As part of the DOM Level 2 specification, the W3C developed their own mechanism for dealing with events. It involves adding an event listener to the DOM element using addEventListiner(type, function, phase). In addition to this, event listeners can be removed using removeEventListener(type, function, phase).

The div below has been given an id called W3C. Multiple event listners have been applied to the div for the onclick event. Refer to the JavaScript to see the code

This div reacts to two different event handlers when the onclick event listener is fired.

JavaScript Code


function cngClass() {
  console.log("Changing Class");
  this.className = "";
  this.className = "alert alert-danger";

}


function cngSize() {
  console.log("Changing Font Size");

  this.style.fontSize = '2em';

}

document.getElementById("W3C").addEventListener('click', cngClass, false);
document.getElementById("W3C").addEventListener("click", cngSize, false);
