Local Storage

HTML5 introduced local storage as a potential replacement for cookies. Cookies are limited in size to 4k. Local storage allows for more storage capacity. W3C guidelines indicate that local storage should be capped at 5Mb.

Storage/Retrieval is achieved by means of a key-value pair. The value is limited to string values. This is considered by many as a severe limitation of the system, in particular in relation to objects. However objects can be stored usuing JSON.stringify(object).

localStorage Object

The localStorge object has the following attributes

localStorage.getItem(key)Get the value associated with key
localStorage.setItem(key, value)Create a new key, value pair
localStorage.removeItem(key)Remove the item associated with key

Form Based Example

In this example the data placed into local storage will be taken from the data in-putted into the form. Each 'key' in the internal storage must be unique. Normally we would never allow users to specify a key at runtime; we would handle this within the code. However in this example, the in-putted key value is tested against all other values and if it is unique the value is entered. If not it is discarded and the user prompted for another 'key-value' pair
