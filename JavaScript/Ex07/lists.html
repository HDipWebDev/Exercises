Dropdown lists can be implemented HTML5. These are handled by the <select> tag in conjunction with a number of nested <option> tags. The <option> tag is also used for the <datalist> shown to the right. Note that the syntax is different in the <datalist> context.

HTML5 introduced the concept of a <datalist>. A datalist is bound to a specific input in the HTML. Each data item is held within an <option> tag. In this case however the syntax is <option value='someValue'>.

When the user enters data into the bound input field, the entered data is checked and highlighted for the user. This functionality is designed to make form filling a little bit easier, but unlike the dropdown implementation, allows for the user to enter a value that has not been previously defined.

It is also worth noting that with autocomplete 'on', your browser will prompt for previously entered values that are not in the datalist. With autocomplete 'off', this does not happen.

Radio and Check

Radio and Checks are also easy to implement in HTML. Note that some of the examples of layout on the net (inc W3C) do not use a bound label. Such practice is to be avoided.

For the most part, implementing both is the same; however the name value for the radio button group must be the same in order for grouping to work. This is not necessary for checkboxes.
