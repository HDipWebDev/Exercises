Part of the problem with the previous event handler is that it involves placing bits of JavaScript into the site HTML. This creates unnecessary complexity in the code and makes maintenance extremly difficult. A better implementation is to keep the event listeners in JavaScript by using:


document.getElementById("someID").onclick.

When the event handler is added in, the final code looks like:


document.getElementById("someID").onclick = doSomething();

Click on this div to see what happens

Also note the scope of the variable 'someText' in the JavaScript file.

JavaScript Code


// This is an example of an anonymous function being used to respond to an external event

document.getElementById("secondFind").onclick = function(){

    var someText = "This is an event handler that has been driven entirely by the JavaScript code.  There are no inline event listeners in the HTML"

    document.getElementById("secondFind").innerHTML = someText;

};
