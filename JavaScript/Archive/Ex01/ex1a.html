JavaScript Cookies

The first browser to use cookie functionality was Netscape. In Computer Science parlance, a cookie is an opaque piece of data ld by an intermediary. In other words it is a small data file on your hard disk (intermediary) that is used by web domains to store some data on your machine

The typical usage of cookies is for holding user information to personalise a website for the user. This can be information such as name, color preferences, etc.

It is generally a bad idea to store any personal information (including name) in a cookie. Cookies do present a risk to users when developers do not adhere to strict data protection controls.

One form of attack (2013) is a 'JavaScript Cookie Bomb'. It works like this:

  • Compromised Site, or mal-intentioned site puts/creates Cookie on your machine
  • Cookie contains <iframe> code that renders into the webpage. The <iframe> content whatever hacker wants it to be, typically an advert of some kind. It could also be an exploit kit or other malware.

A good resource for Cookie information is: cookiecentral.com. Further information on this exploit is also availible at http://malwaremustdie.blogspot.ie/2013/07/oof-of-concept-of-cookiebomb-attack.html

A full, and robust cookie framework is availible from the Mozilla Developer Network at https://developer.mozilla.org/en-US/docs/Web/API/document.cookie
